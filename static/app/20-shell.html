Approval rules

                Per-category gates for the internal agent. Applied live and saved to intendant.toml.
              

              

                
                
              

              

                
                
              

              

                
                
              

              

                
                
              

              

                
                
              

              

                
                
              

              

                
                
              

              

                
                
              

              

                Per-category gates for the internal agent. Auto: run without asking. Ask: prompt for approval. Deny: block outright. Applied live and saved to intendant.toml [approval].
              

            

            

              

                Thread actions
                

                  
                  
                  
                  
                  
                  
                  
                  
                  
                  
                  
                  
                  
                  
                  
                  
                

                

                  Actions dispatch to the live Codex thread. New thread is the only button that takes effect without an active task; typing /fast with no target starts an idle fast Codex session. Goal actions use Codex's experimental persisted goal API. Errors surface as a toast + Activity log entry.
                

              

              

                Sandbox
                

                  
                  
                

                

                  workspace-write blocks listener sockets / pipes and outbound network. Prefer danger-full-access only when a task genuinely needs live app automation (UNO, D-Bus) or arbitrary network.
                

              

              

                Approval policy
                

                  
                  
                

                

                  untrusted: only run known-safe commands without approval. on-request: model asks when it wants to. never: execute without asking (only with an external sandbox).
                

              

              

                Model
                

                  
                  
                

                
Sent as model in thread/start. Empty = let Codex pick its configured default.

                

                  
                  
                

                
Maps to -c model_reasoning_effort=.... Only meaningful for reasoning-capable models; ignored otherwise.

                

                  
                  
                

                

                  Applies to new Intendant-managed Codex sessions. inherit leaves ~/.codex/config.toml and account defaults in charge; normal explicitly clears Fast for these sessions.
                

              

              

                Tools
                

                  
                  
                  off
                

                

                  Enables the Responses API web_search tool (matches codex --search). No per-call approval.
                

                

                  
                  
                

                

                  vanilla keeps upstream/fork Codex behavior. managed enables proactive Intendant context densification, rewind/backout tools, and suppressed Codex auto-compaction for managed long tasks.
                

                

                  
                  
                

                

                  summary keeps compact visualization snapshots. exact persists full raw provider payloads. off disables Context snapshots.
                

              

              

                Sandbox permissions
                

                  
                  
                  off
                

                

                  Only applies when sandbox is workspace-write. Maps to
                  sandbox_workspace_write.network_access. Middle ground between the default and danger-full-access when a task needs outbound HTTP but no listener binds.
                

                

                  
                  
                

                

                  One absolute path per line. Each becomes an entry in
                  sandbox_workspace_write.writable_roots (equivalent to --add-dir). Blank lines and duplicates are dropped. Press Enter-out or click away to save.
                

              

            

            

              

                Model
                

                  
                  
                

                

                  Sent as claude --model. Empty = let the claude CLI pick its configured default. Applies to the next task (the model latches when the process spawns).
                

              

              

                Permission mode
                

                  
                  
                

                

                  default: tools outside the allowed list ask — prompts surface as Intendant approvals. acceptEdits: file edits auto-approve, commands still ask. plan: read-only planning. auto: a classifier auto-approves safe calls, the rest still ask. dontAsk: anything that would ask is denied instead. bypassPermissions: nothing asks — prefer an external sandbox.
                

              

              

                Allowed tools
                

                  
                  
                

                

                  Passed as claude --allowedTools. Listed tools (rule syntax like Bash(cargo test *) works) skip the permission prompt entirely; everything else follows the permission mode above. Press Ctrl/⌘+Enter or click away to save. Applies to the next task.
