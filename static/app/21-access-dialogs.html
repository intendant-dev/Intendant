Credential vault
Provider credentials sealed under passkey/recovery-derived keys. Passive hosted storage receives ciphertext, but code served by that origin can access entries while this page unlocks them. The Connect-account vault has no trusted delivery bridge to a daemon in this alpha; daemon-local vaults, local configuration, and time-boxed leases remain separate paths.
Agent accounts — this daemon
Sign this machine's coding-agent CLIs (Claude Code, Codex) into their accounts without touching a terminal. The daemon drives each official sign-in on a private terminal it never shows to agents; the token exchange stays inside the CLI, and the credential lands in the CLI's own store on this machine. One sign-in ceremony runs at a time.
Custody trail
Every lease grant, expiry, revocation, and browser-relay change this daemon has seen — who, what, and when. Recorded by the daemon itself, kept on its disk, never sent anywhere.
